# ADR-006: 文件上传与存储方案

## 状态
已接受

## 背景
系统需要支持各类电子档案的上传和管理，包括路口/点位的现场照片、验收报告、设备说明书等文件。需要确保文件安全存储、便捷访问和合理组织。

## 决策
采用本地文件系统存储 + 数据库索引管理方案。

### 存储架构

```
uploads/                              # 上传根目录
├── intersection_1_现场照片.jpg
├── intersection_1_验收报告.pdf
├── point_2_设备照片.jpg
└── ...
```

### 文件命名规则

```python
unique_filename = f"{related_entity_type}_{related_entity_id}_{filename}"
```

示例：
- `intersection_1_现场照片.jpg`
- `point_2_验收报告.pdf`

### 支持的文件类型

| 类型 | 扩展名 | 说明 |
|------|--------|------|
| 文档 | pdf, doc, docx | 验收报告、说明书 |
| 表格 | xls, xlsx | 数据表格 |
| 图片 | jpg, jpeg, png, gif | 现场照片 |
| 压缩 | zip, rar | 批量文件（可选） |

### 附件表结构

```python
class Attachment(db.Model):
    related_entity_type: Mapped[str]   # 关联实体类型
    related_entity_id: Mapped[int]    # 关联实体ID
    file_name: Mapped[str]            # 存储文件名
    file_path: Mapped[str]            # 完整路径
    original_filename: Mapped[str]    # 原始文件名
    file_size: Mapped[Optional[int]]  # 文件大小
    mime_type: Mapped[Optional[str]]   # MIME类型
    upload_time: Mapped[datetime]      # 上传时间
    uploaded_by: Mapped[Optional[str]] # 上传者
    description: Mapped[Optional[str]] # 描述
```

### 上传流程

```
1. 用户选择文件
2. 前端验证文件类型和大小
3. POST上传到 /api/attachments/upload
4. 后端验证:
   - 文件类型是否允许
   - 关联实体是否存在
   - 文件是否为空
5. 生成唯一文件名
6. 保存文件到上传目录
7. 创建附件数据库记录
8. 返回上传结果
```

### 安全措施

| 措施 | 说明 |
|------|------|
| 文件类型白名单 | 仅允许预定义的安全类型 |
| 文件名安全处理 | 使用 `secure_filename()` 清理文件名 |
| 唯一文件名 | 避免文件名冲突和覆盖 |
| 大小限制 | 16MB 最大文件大小 |
| 路径隔离 | 上传文件与代码隔离 |

### 文件删除

删除时同时清理：
1. 数据库附件记录
2. 物理文件

```python
def delete(self, attachment_id):
    attachment = db.session.query(Attachment).get(attachment_id)
    if not attachment:
        return {'status': 'error', 'message': '附件不存在'}, 404

    upload_folder = _get_upload_folder()
    file_path = os.path.join(upload_folder, attachment.file_name)
    if os.path.exists(file_path):
        os.remove(file_path)

    db.session.delete(attachment)
    db.session.commit()

    return {'status': 'success', 'message': '删除成功'}
```

### 配置项

```python
# 上传文件夹（绝对路径）
UPLOAD_FOLDER = os.path.abspath(os.environ.get('UPLOAD_FOLDER', 'uploads'))

# 最大文件大小（16MB）
MAX_CONTENT_LENGTH = 16 * 1024 * 1024

# 允许的扩展名
ALLOWED_EXTENSIONS = {'pdf', 'doc', 'docx', 'xls', 'xlsx', 'jpg', 'jpeg', 'png', 'zip', 'rar'}
```

## 替代方案考虑

| 替代方案 | 优点 | 缺点 |
|---------|------|------|
| 云存储(OSS/S3) | 弹性扩展，高可用 | 需第三方服务，增加成本 |
| 数据库BLOB | 数据一致性高 | 数据库膨胀，备份困难 |
| FTP服务器 | 传统方案，兼容性好 | 配置复杂，权限管理繁琐 |

## 后果

### 正面
- 实现简单，无外部依赖
- 文件直链访问，性能好
- 文件与数据库记录分离，便于管理
- 统一的附件管理，支持多实体关联

### 负面
- 单机存储，无分布式支持
- 需要自行处理备份和冗余
- 文件迁移需要手动处理
- 不适合海量文件场景

## 未来优化方向

1. **云存储迁移**：可平滑迁移到阿里云OSS或AWS S3
2. **CDN加速**：静态资源使用CDN分发
3. **文件压缩**：大文件自动压缩
4. **缩略图**：图片自动生成缩略图
5. **增量备份**：定期备份上传目录
