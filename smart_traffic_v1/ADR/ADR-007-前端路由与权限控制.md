# ADR-007: 前端路由与权限控制

## 状态
已接受

## 背景
系统采用 Vue 3 + Vue Router 4 构建 SPA 应用，需要实现页面路由控制和基于角色的访问权限管理。用户登录后能访问对应权限的页面，未登录用户需要重定向到登录页。

## 决策
采用 Vue Router 4 的路由守卫 + Pinia 状态管理方案。

### 路由结构设计

```typescript
const routes: RouteRecordRaw[] = [

```

```
├── /parking-enforcements → ParkingEnforcementList

├── /checkpoints/:id     → CheckpointDetail

```

### 路由守卫实现

```typescript
router.beforeEach(async (to, _from, next) => {
  const userStore = useUserStore()

  // 1. 如果本地有Token但Pinia中无用户信息，尝试初始化
  if (!userStore.user && (localStorage.getItem('token') || sessionStorage.getItem('token'))) {
    userStore.initFromStorage()
    await userStore.fetchCurrentUser()
  }

  // 2. 检查是否需要登录
  if (to.meta.requiresAuth && !userStore.isLoggedIn) {
    next('/login')
  }
  // 3. 已登录用户访问登录页则重定向到首页
  else if (to.path === '/login' && userStore.isLoggedIn) {
    next('/')
  }
  // 4. 其他情况放行
  else {
    next()
  }
})
```

### 角色权限定义

| 角色 | 权限说明 |
|------|----------|
| admin | 管理员：全部权限 |
| editor | 编辑：创建、编辑、查看 |
| viewer | 查看者：仅查看 |

### 前端权限控制策略

| 功能 | admin | editor | viewer |
|------|-------|--------|--------|
| 查看列表 | ✅ | ✅ | ✅ |
| 查看详情 | ✅ | ✅ | ✅ |
| 创建 | ✅ | ✅ | ❌ |
| 编辑 | ✅ | ✅ | ❌ |
| 删除 | ✅ | ❌ | ❌ |
| 统计分析 | ✅ | ✅ | ✅ |

### 按钮级别权限控制

```vue
<template>
  <el-button v-if="hasPermission('create')" type="primary">
    创建
  </el-button>
  <el-button v-if="hasPermission('delete')" type="danger">
    删除
  </el-button>
</template>

<script setup>
const hasPermission = (action) => {
  const role = userStore.userRole
  const permissions = {
    admin: ['create', 'edit', 'delete', 'view'],
    editor: ['create', 'edit', 'view'],
    viewer: ['view']
  }
  return permissions[role]?.includes(action) || false
}
</script>
```

### Token 持久化策略

```typescript
// 登录成功后存储Token
localStorage.setItem('token', response.data.token)
sessionStorage.setItem('token', response.data.token)

// 应用启动时恢复
if (localStorage.getItem('token')) {
  userStore.initFromStorage()
  await userStore.fetchCurrentUser()
}
```

### 页面刷新处理

```
页面刷新
    ↓
检查 localStorage/sessionStorage 是否有Token
    ↓
有Token → 调用 /api/auth/current_user 获取用户信息 → 进入目标页面
    ↓
无Token → 重定向到 /login
```

## 替代方案考虑

| 替代方案 | 优点 | 缺点 |
|---------|------|------|
| 路由元信息+守卫 | 官方推荐，简单直接 | 权限控制粒度粗 |
| 动态路由生成 | 可根据权限动态添加路由 | 实现复杂 |
| 组件内权限控制 | 粒度细 | 代码分散 |

## 后果

### 正面
- 路由守卫统一拦截，管理清晰
- Pinia状态管理，用户信息全局共享
- 页面刷新自动恢复登录状态
- 嵌套路由结构清晰

### 负面
- 权限控制在前端，后端需配合验证
- Token存储在本地可能有XSS风险
- 路由配置相对固定
