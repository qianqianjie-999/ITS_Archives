# 智能交通建设档案系统数据库设计文档

## 1. 概述

本数据库用于管理智能交通领域的前端设备（信号灯、电子警察、违停球、卡口）、后端设备、项目信息、质保状态及电子档案。支持一个路口/点位关联多个项目（新建、续保等），并自动计算当前质保状态。

**数据规模**：约 500 个路口，总记录数在万级以内，采用 MySQL 数据库即可轻松支撑。

## 2. 核心设计原则

- 每个设备表（信号灯、电警等）都关联 `project_id`，保留项目信息。
- 路口和点位作为独立实体，设备表通过外键引用。
- 续保（无新增设备）单独使用 `warranty_extension` 表记录，避免产生空设备记录。
- 使用视图实时计算每个路口/点位的质保状态（基于关联的所有项目中最晚的质保到期时间）。
- 附件表统一存储各类电子档案，可关联任意实体。

## 3. 表结构

### 3.1 项目表（project）

存储所有项目的通用信息。

```sql
CREATE TABLE project (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '项目名称',
    acceptance_date DATE COMMENT '验收日期',
    warranty_period VARCHAR(50) COMMENT '质保期（如 2年）',
    warranty_expire_date DATE NOT NULL COMMENT '质保到期时间',
    builder VARCHAR(100) COMMENT '建设单位',
    constructor VARCHAR(100) COMMENT '施工单位'
);
```

### 3.2 路口表（intersection）

路口基础信息，用于信号灯和电子警察。

```sql
CREATE TABLE intersection (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '路口名称',
    type ENUM('十字路口', '丁字路口', '行人过街', '其他') COMMENT '路口类型'
);
```

### 3.3 点位表（point）

点位基础信息，用于违停球和卡口。

```sql
CREATE TABLE point (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '点位名称',
    area VARCHAR(100) COMMENT '抓拍区域（违停球用）',
    type VARCHAR(50) COMMENT '卡口类型或违停球类型'
);
```

### 3.4 信号灯表（traffic_light）

记录路口信号灯设备的配置。

```sql
CREATE TABLE traffic_light (
    id INT PRIMARY KEY AUTO_INCREMENT,
    intersection_id INT NOT NULL COMMENT '路口ID',
    project_id INT NOT NULL COMMENT '所属项目ID',
    signal_type VARCHAR(50) COMMENT '信号机类型（智能/非智能）',
    signal_count INT DEFAULT 0,
    left_arrow_count INT DEFAULT 0,
    straight_arrow_count INT DEFAULT 0,
    right_arrow_count INT DEFAULT 0,
    full_screen_count INT DEFAULT 0,
    non_motor_count INT DEFAULT 0,
    pedestrian_count INT DEFAULT 0,
    radar_count INT DEFAULT 0,
    guide_screen_count INT DEFAULT 0,
    power_source TEXT COMMENT '取电说明',
    FOREIGN KEY (intersection_id) REFERENCES intersection(id),
    FOREIGN KEY (project_id) REFERENCES project(id)
);
```

### 3.5 电子警察表（electronic_police）

记录路口电子警察设备配置。

```sql
CREATE TABLE electronic_police (
    id INT PRIMARY KEY AUTO_INCREMENT,
    intersection_id INT NOT NULL,
    project_id INT NOT NULL,
    capture_type VARCHAR(50) COMMENT '抓拍类型',
    terminal_server_count INT DEFAULT 0,
    forward_capture_count INT DEFAULT 0,
    reverse_capture_count INT DEFAULT 0,
    led_light_count INT DEFAULT 0,
    strobe_light_count INT DEFAULT 0,
    ptz_count INT DEFAULT 0,
    signal_detector_count INT DEFAULT 0,
    network_source TEXT COMMENT '取网说明',
    FOREIGN KEY (intersection_id) REFERENCES intersection(id),
    FOREIGN KEY (project_id) REFERENCES project(id)
);
```

### 3.6 违停球表（parking_enforcement）

记录违停球点位设备配置。

```sql
CREATE TABLE parking_enforcement (
    id INT PRIMARY KEY AUTO_INCREMENT,
    point_id INT NOT NULL,
    project_id INT NOT NULL,
    camera_count INT DEFAULT 0,
    parking_sign_count INT DEFAULT 0,
    monitor_sign_count INT DEFAULT 0,
    power_source TEXT,
    network_source TEXT,
    FOREIGN KEY (point_id) REFERENCES point(id),
    FOREIGN KEY (project_id) REFERENCES project(id)
);
```

### 3.7 卡口表（checkpoint）

记录卡口点位设备配置。

```sql
CREATE TABLE checkpoint (
    id INT PRIMARY KEY AUTO_INCREMENT,
    point_id INT NOT NULL,
    project_id INT NOT NULL,
    camera_count INT DEFAULT 0,
    strobe_light_count INT DEFAULT 0,
    radar_count INT DEFAULT 0,
    sign_count INT DEFAULT 0,
    power_source TEXT,
    network_source TEXT,
    FOREIGN KEY (point_id) REFERENCES point(id),
    FOREIGN KEY (project_id) REFERENCES project(id)
);
```

### 3.8 后端设备表（backend_device）

记录机房/后端设备，不直接关联路口或点位。

```sql
CREATE TABLE backend_device (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    type VARCHAR(50) COMMENT '设备类型（网络交换设备、服务器等）',
    project_id INT NOT NULL,
    FOREIGN KEY (project_id) REFERENCES project(id)
);
```

### 3.9 续保记录表（warranty_extension）

用于无新增设备、仅延长质保的场景。通过该表将路口或点位与续保项目关联。

```sql
CREATE TABLE warranty_extension (
    id INT PRIMARY KEY AUTO_INCREMENT,
    facility_type ENUM('intersection', 'point') NOT NULL COMMENT '设施类型',
    facility_id INT NOT NULL COMMENT '路口ID或点位ID',
    project_id INT NOT NULL COMMENT '续保项目ID',
    extension_date DATE NOT NULL COMMENT '续保生效日期',
    FOREIGN KEY (project_id) REFERENCES project(id)
);
```

### 3.10 附件表（attachment）

统一存储所有电子档案（验收报告、施工图、合同扫描件等），可关联任意实体。

```sql
CREATE TABLE attachment (
    id INT PRIMARY KEY AUTO_INCREMENT,
    related_entity_type ENUM('project', 'intersection', 'point', 'traffic_light', 'electronic_police', 'parking_enforcement', 'checkpoint', 'backend_device') NOT NULL,
    related_entity_id INT NOT NULL COMMENT '对应表的记录ID',
    file_name VARCHAR(255) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    file_size INT COMMENT '字节数',
    mime_type VARCHAR(100),
    upload_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    uploaded_by VARCHAR(100),
    description TEXT
);
```

## 4. 视图：质保状态自动计算

### 4.1 路口质保状态视图（intersection_warranty_status）

收集信号灯、电子警察、续保表中该路口关联的所有项目，取最晚的质保到期时间，与当前日期比较得出状态。

```sql
CREATE VIEW intersection_warranty_status AS
WITH all_intersection_projects AS (
    SELECT intersection_id, project_id FROM traffic_light
    UNION
    SELECT intersection_id, project_id FROM electronic_police
    UNION
    SELECT facility_id AS intersection_id, project_id FROM warranty_extension WHERE facility_type = 'intersection'
)
SELECT 
    i.id,
    i.name,
    MAX(p.warranty_expire_date) AS latest_expire_date,
    CASE 
        WHEN MAX(p.warranty_expire_date) >= CURDATE() THEN '在保'
        WHEN MAX(p.warranty_expire_date) IS NULL THEN '无项目'
        ELSE '过保'
    END AS warranty_status
FROM intersection i
LEFT JOIN all_intersection_projects aip ON i.id = aip.intersection_id
LEFT JOIN project p ON aip.project_id = p.id
GROUP BY i.id, i.name;
```

### 4.2 点位质保状态视图（point_warranty_status）

收集违停球、卡口、续保表中该点位关联的所有项目，计算质保状态。

```sql
CREATE VIEW point_warranty_status AS
WITH all_point_projects AS (
    SELECT point_id, project_id FROM parking_enforcement
    UNION
    SELECT point_id, project_id FROM checkpoint
    UNION
    SELECT facility_id AS point_id, project_id FROM warranty_extension WHERE facility_type = 'point'
)
SELECT 
    p.id,
    p.name,
    p.area,
    MAX(proj.warranty_expire_date) AS latest_expire_date,
    CASE 
        WHEN MAX(proj.warranty_expire_date) >= CURDATE() THEN '在保'
        WHEN MAX(proj.warranty_expire_date) IS NULL THEN '无项目'
        ELSE '过保'
    END AS warranty_status
FROM point p
LEFT JOIN all_point_projects app ON p.id = app.point_id
LEFT JOIN project proj ON app.project_id = proj.id
GROUP BY p.id, p.name, p.area;
```

## 5. 常用查询示例

### 5.1 查询所有路口及其当前质保状态

```sql
SELECT * FROM intersection_warranty_status;
```

### 5.2 查询某个路口的详细设备信息

```sql
SELECT 
    i.name,
    tl.signal_type,
    tl.signal_count,
    tl.pedestrian_count,
    p.name AS project_name,
    p.warranty_expire_date
FROM intersection i
LEFT JOIN traffic_light tl ON i.id = tl.intersection_id
LEFT JOIN project p ON tl.project_id = p.id
WHERE i.id = 1;
```

### 5.3 为路口续保（无新增设备）

```sql
-- 插入新项目
INSERT INTO project (name, acceptance_date, warranty_period, warranty_expire_date)
VALUES ('2026年续保项目', '2026-01-01', '1年', '2026-12-31');

-- 关联续保记录
INSERT INTO warranty_extension (facility_type, facility_id, project_id, extension_date)
VALUES ('intersection', 1, LAST_INSERT_ID(), '2026-01-01');
```

### 5.4 上传附件（例如为项目上传验收文件）

```sql
INSERT INTO attachment (related_entity_type, related_entity_id, file_name, file_path, mime_type)
VALUES ('project', 1, '验收报告.pdf', '/uploads/project_1/acceptance.pdf', 'application/pdf');
```

### 5.5 查询某个项目的所有附件

```sql
SELECT * FROM attachment WHERE related_entity_type = 'project' AND related_entity_id = 1;
```

## 6. 性能与维护建议

- **索引**：建议在以下字段上创建索引（数据量虽小，但养成好习惯）：
  - `project(warranty_expire_date)`
  - `traffic_light(project_id)`, `traffic_light(intersection_id)`
  - `electronic_police(project_id)`, `electronic_police(intersection_id)`
  - `warranty_extension(facility_type, facility_id)`
  - `attachment(related_entity_type, related_entity_id)`
- **视图实时计算**：由于数据量小（500+路口），视图查询每次均为毫秒级，无需物化。
- **数据完整性**：虽然 `warranty_extension` 没有跨表外键，但应用层需保证 `facility_id` 存在于对应的路口或点位表中。
- **历史记录**：如需保留设备配置变更历史，可添加 `version` 字段或使用日志表，当前设计直接更新即可覆盖最新状态（每个项目一条设备记录）。

## 7. 扩展可能性

- 增加区域/网格表，关联到路口和点位，实现分级管理。
- 增加设备状态表（在线/离线/故障），实时监控。
- 将视图结果定时物化到普通表，供大屏或报表系统使用（当前规模无需此操作）。
